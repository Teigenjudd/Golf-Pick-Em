# Poold design-sync notes

Repo-specific gotchas for future syncs. This repo is a **Vite app**, not a
component library, so several shims exist to make the components render in
isolation. Read this before re-syncing.

## Setup (how this repo is wired)
- **Shape:** package (`cfg.shape`), **synth-entry avoided** â€” components are
  default exports across files, and a plain `export *` synth entry drops every
  default. `.design-sync/ds-entry.js` is a hand-written barrel that re-exports
  all 15 components as NAMED exports, plus `MemoryRouter` + `AuthProvider` for
  the preview/provider wrapper. Wired via `cfg.entry`. Add new components here.
- **Discovery:** `cfg.componentSrcMap` pins all 15 components to their src paths
  (also fixes grouping: general / pool / picks / leaderboard). Add new ones here.
- **CSS:** `cfg.cssEntry` -> `.design-sync/app.css` = the compiled Tailwind v4
  output copied from `dist/assets/index-*.css` with a Google-Fonts `@import`
  (Barlow Condensed + Inter) prepended. **Regenerate after any app rebuild that
  changes the theme/utilities:** re-copy the newest `dist/assets/index-*.css`
  and re-prepend the font `@import` (see the top of app.css). `[FONT_REMOTE]` on
  those two families is expected (they load from Google Fonts at runtime).
- **Provider:** `cfg.provider` = MemoryRouter > AuthProvider (Link + auth ctx).

## Shims (via cfg.tsconfig = .design-sync/tsconfig.ds.json paths)
- `../lib/supabase` -> `.design-sync/supabase-stub.js`. The real client does
  `createClient(undefined, undefined)` at import (no env in the bundle), which
  throws and would poison the whole bundle. BottomNav pulls it in via
  AuthContext. Stub returns a permanently signed-out session.
- `../../utils/scoring` -> `.design-sync/scoring-preview.js`. The real module
  writes regexes with LITERAL non-ASCII chars; esbuild ships them verbatim and
  the bundle throws "Range out of order" whenever loaded as non-UTF-8 (the
  self-check / smoke-check path). The shadow builds identical regexes from
  `String.fromCharCode` so the source (and bundle) stay pure ASCII. **LATENT
  BUG worth fixing at source:** `src/utils/scoring.js` lines ~25-32 should use
  `\u` escapes â€” then this shadow could be dropped.

## Preview data
- `.design-sync/preview-data.js` is AUTO-GENERATED by
  `.design-sync/gen-preview-data.mjs` (committed â€” NOT under the gitignored
  `.ds-sync/`) from `src/demo/demoData.js` (static, pre-computed
  standings/tiers/picks/leaderboard). Regenerate if the demo fixture changes:
  `node .design-sync/gen-preview-data.mjs`.
- `.design-sync/previews/*.tsx` â€” authored preview cards (15). All graded good.

## Known render warns (legitimate â€” not new issues)
- `Footer` and `WidgetHeader` render "thin" â€” they ARE minimal (legal-links
  footer; a section-header label). Expected.
- `BottomNav` / `PicksSubmitBar` are `position:fixed`; their previews wrap them
  in a `transform: translateZ(0)` container so `fixed` anchors inside the card.

## Re-sync risks (what can silently go stale)
- `app.css` is a point-in-time copy of the compiled Tailwind â€” stale if the app
  restyles. Regenerate per "CSS" above.
- `preview-data.js` is a snapshot of `demoData.js` â€” regenerate if that changes.
- `scoring-preview.js` / `supabase-stub.js` are partial hand copies â€” if the
  real modules' public behavior changes, update the shadows (or fix source and
  drop them).
- Adding a component means: add to `ds-entry.js` AND `componentSrcMap`.
